    Bootstrap Folder Structure Plan¶

    Purpose of this file: This document is the master plan for the bootstrap/ section of this runbook. It defines every subfolder, every file, and maps every tool from the install-tools-all.sh script (31 phases, 350+ tools) to its exact location inside bootstrap/.

    Golden Rule: bootstrap/ contains installation runbooks only. How to use a tool after it is installed belongs in its respective operational section (kubernetes/, delivery/, security/, observability/, etc.).

    What is bootstrap/?¶

    bootstrap/ answers one and only one question:

    "I have a fresh Ubuntu 24.04 machine. How do I install <tool> on it?"

    Every file inside bootstrap/ is an installation runbook — the commands, the method (apt, pip, cargo, binary, script), the version pinning, and the PATH/environment setup required to make the tool available system-wide.

    This section is derived from the master script: iximiuz/rootfs/dev/machine/scripts/install-tools-all.sh
